🦴 01 Section 速查表

1. 核心段落定义 (7 Section Types, 8 维度含 Section 顺序)

H (Hook) 注意力捕获打断自动滑动，争夺认知带宽（必需）
Br (Bridge) 关联建立建立语境与个人相关性（可省略）
P (Problem) 张力建立暴露现状与理想差距（强烈建议）
Re (Resolution) 张力释放缓解张力，展示改善路径（极高建议）
Rv (Reversal) 认知改写推翻既有认知，消除潜在抗拒
V (Value) 决策稳定产品/服务的核心价值（强烈建议）
C (CTA) 行为触发将选择倾向转化为行动（必需）

2. 常见 Skeleton 组合 (8 种)

类型结构流Tier时长比例参考适用场景
HPVCH → P → V → CA15%/25%/45%/15%标准四段，常规种草
HBVCH → Br → V → CA15%/20%/50%/15%带桥接的快速种草
HRVCH → Re → V → CA15%/25%/45%/15%带解决方案的直接种草
HPRVCH → P → Rv → V → CB10%/20%/20%/35%/15%万能五段，适合种草、产品介绍
HPBVCH → P → Br → V → CB10%/20%/15%/40%/15%问题后桥接，强化相关性
HBrPVCH → Br → P → V → CB10%/15%/20%/40%/15%先桥接再问题，建立信任
HBrRVCH → Br → Rv → V → CB10%/15%/25%/35%/15%桥接加认知改写，突破固有认知
HBrPReVCH → Br → P → Re → V → CC8%/12%/18%/17%/30%/15%完整六段，深度内容论述

Tier A = 4段以内（短平快）; Tier B = 5段（标准说服）; Tier C = 6段+（深度论述）

字段类型说明

beauty变美

4. Conflict Type (冲突类型 5 枚举)

internal内心矛盾

5. Emotion Arc (情感弧线 — 节点数组)

每个 Section 对应一个情绪节点，标注该段的情绪标签与强度值（0-10）。整条弧线由节点数组 nodes[] 组成。

字段类型说明
sectionenum (H/Br/P/Re/Rv/V/C)所属段落
emotionstring该段主要情绪标签（如 curiosity, anxiety, relief, desire, urgency）
intensitynumber 0-10情绪强度值，0 = 平静，10 = 极端

示例：[{section:"H", emotion:"curiosity", intensity:7}, {section:"P", emotion:"anxiety", intensity:8}, {section:"V", emotion:"desire", intensity:9}, {section:"C", emotion:"urgency", intensity:6}]

🥩 04 Meat 速查表

4 个分组、11 个字段 — 从基础格式到增强元素，层叠定义视频的视觉表现。

分组一：格式约束 (Format) — 3 字段

1.1 出镜方式 talent_framing

#Code中文名描述
1product_only纯产品无人出镜，仅展示产品本体；适合产品特写、ASMR 开箱、成分展示
2hands_demo手部演示手部出镜操作产品，不露脸；适合教程步骤、上妆演示
3body_no_face身体不露脸身体可见但不展示面部；适合穿搭展示、身材对比
4talking_head口播面部出镜，对镜讲解；人设感强，适合测评、种草、知识分享
5full_body全身出镜全身入镜，动作表演；适合穿搭、健身、生活 vlog
6skit情景剧多人演绎，剧情表演；传播潜力大，适合搞笑段子、反转剧情

1.2 时长范围 duration_range

子字段类型描述
min_secondsnumber最短时长（秒）— 低于此时长信息无法完整传递
max_secondsnumber最长时长（秒）— 超过此时长完播率会显著下降
target_secondsnumber目标时长（秒）— 最理想的时长，平衡信息量和完播率

1.3 画面比例 aspect_ratio

9:16竖版 — 抖音/快手/Reels
16:9横版 — YouTube/B站
1:1方形 — 信息流/商品展示

分组二：视听风格 (Audiovisual) — 4 字段

2.1 语音类型 voice_type

#Code中文名描述
1human真人口播/配音信任感最强，适合需要人设和情感传递的内容
2ai_ttsAI 配音制作效率高、可批量生成，适合信息密集型内容
3none无语音纯视觉 + 文字，适合 ASMR、产品展示、纯视觉冲击类

2.2 镜头语法 shot_grammar

子字段类型描述
section_constraintsobject每个 Section 段落的镜头数量约束：min_shots / max_shots / min_duration_ratio
rulesstring[]构图规则列表，如 "三分法" / "黄金比例" / "中心对称"

2.3 运镜 camera_movement

#Code中文名描述
1static固定镜头锁定不动；稳定感强，适合口播、产品特写
2push推向前推进，聚焦细节；制造紧迫感和关注度
3pull拉向后拉远，展示全貌；制造空间感和揭示感
4pan横摇水平方向左右移动；适合展示宽幅场景、对比展示
5tilt竖摇垂直方向上下移动；适合展示纵向物体（全身穿搭）
6track移/跟跟随主体运动；动态感最强，营造沉浸式体验

2.4 字幕形式 subtitle_form

#Code中文名描述
1punchline金句型大字居中，强调核心金句；适合卖点提炼、痛点放大
2steps步骤型1.2.3. 步骤编号，教程感；适合使用方法、操作流程
3label标签型小字标注，辅助说明；不抢画面，适合产品名称、成分标注
4data_card数据卡片型数据可视化呈现；适合实验数据、销量、对比百分比
5minimal极简几乎无字幕，画面优先；适合视觉冲击类、ASMR

分组三：节奏 & 剪辑 (Rhythm) — 2 字段

3.1 节奏 pacing

level — 节奏等级：

#值中文名描述
1low慢节奏长镜头为主，沉浸感强；适合故事叙事、品牌调性、ASMR
2medium中节奏均衡节奏，适中切换；大多数场景的安全选择
3high快节奏快速剪辑，信息密度高；适合促销带货、快速种草

shot_duration — 每镜时长约束：

参数中文名描述
min最短秒数单镜头最短秒数 — 低于此观众来不及接收信息
max最长秒数单镜头最长秒数 — 超过此画面会显得拖沓
avg平均秒数整条视频镜头时长的期望均值，剪辑的核心参考

3.2 剪辑模式 editing_pattern

style — 剪辑风格文字描述（如 "快速跳切+节奏卡点" / "平稳叙事型剪辑"）

transitions — 允许使用的转场方式：

#值中文名描述
1hard_cut硬切镜头直接切换；节奏感强，信息传递效率高
2dissolve溶解前后画面渐变过渡；柔和自然，适合情绪转换
3effects特效转场缩放、旋转、闪白等；视觉冲击力强，不宜过多

分组四：增强元素 (Enhance) — 2 字段

4.1 屏幕元素 screen_elements

allowed_types — 允许使用的元素类型：

#值中文名描述
1花字花字加粗强调文字；突出关键卖点、金句、数据
2emojiEmojiemoji 装饰；增加趣味性和亲和力
3data_callout数据标注百分比、数字、图表等数据可视化，增强说服力
4arrow箭头引导观众视线聚焦到画面特定区域
5sticker贴纸装饰性元素，增加画面丰富度
6broll_insertB-roll补充画面素材插入，丰富视觉层次和叙事节奏
hook_required — HOOK 段是否必须添加屏幕元素（花字/视觉元素提升前 1-2 秒停留率）
cta_required — CTA 段是否必须添加屏幕元素（箭头/按钮提示降低操作犹豫）
max_text_length — 单条花字最大字数（控制简洁度）
rules — 额外规则，如 "每段最多2个花字" / "B-roll 不超过总时长20%"

4.2 音效指导 audio_guidance

bgm_style — BGM 风格：

#值中文名描述
1upbeat欢快节奏明快；适合快节奏种草、促销带货
2emotional情感抒情感人；适合故事叙事、品牌片、用户见证
3minimal极简低调背景；不抢注意力，适合口播为主的内容
4none无 BGM不使用背景音乐；适合 ASMR、追求最大化真实感

sfx_density — 音效使用密度：

#值中文名描述
1high高密度几乎每个镜头切换都有音效；适合快节奏、高能量
2medium中密度关键节点加音效；重要转折和强调处点缀
3low低密度偶尔点缀；只在最关键时刻使用，保持整体克制
4none无音效不使用任何音效；适合纯口播、ASMR

sfx_guidelines — 额外音效规则列表（如 "HOOK 段必须有注意力提示音" / "转场使用 whoosh 音效"）

Meat Profile 不包含的内容： 妆发服（取决于具体演员和产品调性）、拍摄场景（取决于实际拍摄资源）、产品信息（属于具体选题的业务数据）。简言之：Meat 定义"用什么风格拍"，而非"拍什么"。
